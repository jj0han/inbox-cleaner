## Phase 1: Foundation & Authentication
**Goal:** Establish secure connection to Gmail API and fetch necessary metadata.
**Requirements:** AUTH-01, AUTH-02
**Success Criteria:**
- [x] User can log in with Google account and grant necessary permissions.
- [x] App can fetch a high-level summary of the inbox (e.g., total message count).
- [x] No message bodies are downloaded or stored.

## Phase 2: UI Dashboard & Smart Preview
**Goal:** Build the core Notion-style interface and preview capability to establish trust.
**Requirements:** DASH-01, DASH-02, SAFE-01
**Success Criteria:**
- [x] Dashboard displays Hero section with total cleanable emails.
- [x] Action cards render dynamically based on available automations.
- [x] Clicking an action card opens a preview modal with 5-10 example emails.

## Phase 3: Core Automations & Undo
**Goal:** Implement the primary 1-click cleanup actions and the safety undo buffer.
**Requirements:** AUTO-01, AUTO-02, AUTO-03, SAFE-02
**Success Criteria:**
- [x] User can successfully execute "Mass Unsubscribe" or "Archive Promos" via action cards.
- [x] User can execute the "Intelligent 1-Click Cleanup" for comprehensive cleaning.
- [x] Any action triggers a 10-30 second undo toast; clicking it fully reverts the action.

## Phase 4: Smart Rules & Polish
**Goal:** Add AI-lite suggestions and finalize the application experience.
**Requirements:** AUTO-04
**Success Criteria:**
- [x] Application suggests auto-archive rules based on unread patterns.
- [x] User can apply these rules with a single click.
- [x] Undo reverses the archive AND deletes the Gmail filter.
- [x] Action cards show live email count badges.
- [x] Dashboard inbox count reflects actual INBOX size.
- [x] Mobile layout stacks correctly at small viewports.

## Milestone Summary

### Key Decisions
| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Next.js + tRPC | Type-safe full-stack with zero boilerplate | ✓ Good — developer velocity high |
| Prisma 7 + SQLite | Zero-config local DB for undo log state | ✓ Good — worked perfectly for phase 3 |
| Gmail batchModify | Single API call archives up to 1000 messages | ✓ Good — fast and reliable |
| gmail.settings.basic scope | Required for filter create/delete | ✓ Good — re-auth notice documented |
| resultSizeEstimate for counts | Avoids full list pagination | ✓ Good — performant approximation |

### Technical Debt
- `getPreview` still returns hardcoded mock data (not real Gmail preview)
- `getCardCounts` runs 4 parallel Gmail list calls — could be slow if rate-limited
- No error boundary for `SuggestionsSection` failures
