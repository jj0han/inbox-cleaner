## Milestone Summary

**Key Decisions:**
- **Exponential Backoff**: Used 500ms -> 1s -> 2s backoff for 429/503 errors.
- **Server-side Mailto**: Decided to send unsubscribe emails from the server via Gmail API to avoid client-side "mailto:" popup friction.
- **tRPC Error Surfacing**: Switched from swallowing errors in `getSuggestions` to throwing `TRPCError` to leverage React Query's built-in error handling.

**Issues Resolved:**
- Fixed bug where `getSuggestions` swallowed errors, masking failure from the user.
- Closed resilience gap in bulk operations (missing `withRetry`).
- Resolved missing verification artifacts for early v1.1 phases.

**Technical Debt Incurred:**
- **Serial Unsubscribe**: Bulk unsubscribe is serial (1 email at a time) to avoid quota hits; might be slow for very large batches.
